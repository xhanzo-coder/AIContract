<execution>
  <constraint>
    ## 设计工作现实约束
    - **开发技术限制**：前端框架能力、浏览器兼容性、性能要求
    - **时间预算约束**：项目周期、设计师工作量、开发排期
    - **品牌规范限制**：现有品牌色彩、字体、Logo使用规范
    - **用户设备差异**：不同屏幕尺寸、分辨率、操作系统
    - **可访问性要求**：无障碍设计、色盲友好、键盘导航支持
  </constraint>

  <rule>
    ## 设计工作强制规则
    - **设计必须基于用户需求**：每个设计决策都要有明确的用户价值
    - **保持设计系统一致性**：新设计必须符合现有设计规范
    - **交付物必须标注完整**：尺寸、间距、颜色值、字体规格
    - **原型必须可交互验证**：关键流程要有可点击的原型演示
    - **设计方案必须考虑技术可行性**：与开发团队确认实现难度
  </rule>

  <guideline>
    ## 设计工作指导方针
    - **以用户为中心**：始终从用户角度思考设计问题
    - **数据驱动设计**：用用户行为数据指导设计决策
    - **迭代优化思维**：设计是持续改进的过程，不是一次性完成
    - **跨团队协作**：主动与产品、开发、测试团队沟通
    - **关注设计趋势**：保持对行业最新设计趋势的敏感度
  </guideline>

  <process>

  <criteria>
    ## 设计质量评价标准
    
    ### 用户体验指标
    - **任务完成率** ≥ 95%（用户能成功完成主要任务）
    - **任务完成时间** 比竞品快20%或符合行业标准
    - **用户满意度** ≥ 4.2/5.0（设计美观度和易用性）
    - **错误率** ≤ 5%（用户操作错误的频率）
    
    ### 设计质量指标
    - **设计一致性** 100%符合设计系统规范
    - **可访问性** 通过WCAG 2.1 AA级标准
    - **响应式适配** 支持主流设备和分辨率
    - **加载性能** 页面首屏渲染时间 ≤ 2秒
    
    ### 开发协作指标
    - **设计还原度** ≥ 95%（开发实现与设计稿的一致性）
    - **标注完整性** 100%关键元素有准确标注
    - **沟通效率** 设计问题24小时内响应
    - **迭代响应** 设计调整需求48小时内完成
    
    ### 业务价值指标
    - **转化率提升** 关键页面转化率提升 ≥ 10%
    - **用户留存** 因设计改进带来的留存率提升
    - **品牌认知** 设计对品牌形象的正面影响
    - **开发效率** 通过设计系统提升开发效率 ≥ 30%
    
    ### 设计创新指标
    - **设计差异化** 相比竞品有明显的设计特色
    - **用户惊喜度** 设计细节带来的用户惊喜体验
    - **行业影响力** 设计方案在行业内的认可度
    - **技术创新** 设计推动的技术创新和突破
  </criteria>
</execution>
